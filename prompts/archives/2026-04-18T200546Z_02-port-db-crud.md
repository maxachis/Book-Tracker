# Port Book & Settings CRUD to Go

## Goal

Move every database operation currently in `wails/frontend/src/lib/services/database.ts` from direct SQL (`@tauri-apps/plugin-sql`) into Go methods on `App`, backed by the existing `store.Store`. Each method is test-driven: write a failing `store` test first, implement, then expose on `App`.

## Context

- `wails/store/store.go` opens a SQLite DB via `modernc.org/sqlite` and runs embedded migrations, but has zero CRUD methods.
- `wails/service/service.go` holds a `*store.Store` pointer; real methods live in `wails/service/books.go` (currently only `GenerateBookID`, `ValidateCreateBookRequest`).
- `wails/model/book.go` and `settings.go` define `Book`, `CreateBookRequest`, `UpdateBookRequest`, `UserSettings`, `UpdateSettingsRequest` with JSON tags that must match the TypeScript `types/index.ts` shapes.
- `wails/migrations/schema.sql` has the canonical table shape.

## Methods to port

For each method: (a) add a `*_test.go` test in `store/` that exercises it against `store.Open(":memory:")`; (b) implement in `store/`; (c) wrap on `*service.Service` if there's business logic (completion detection, defaults), otherwise call `Store` directly from `App`; (d) expose on `App` in `app.go` with the exact name.

| Frontend caller | New Go method on `App` | Notes |
|---|---|---|
| `createBook` | `CreateBook(req model.CreateBookRequest) (model.Book, error)` | Generate `id`, set `created_at=now()`, `current_progress=0`, `completed_at=nil`. Validate first. |
| `getActiveBooks` | `ListActiveBooks() ([]model.Book, error)` | `WHERE completed_at IS NULL ORDER BY created_at DESC`. |
| `getCompletedBooks` | `ListCompletedBooks() ([]model.Book, error)` | `WHERE completed_at IS NOT NULL ORDER BY completed_at DESC`. |
| `getAllBooks` | `ListAllBooks() ([]model.Book, error)` | Primarily used by CSV export. |
| `updateBook` | `UpdateBook(req model.UpdateBookRequest) (model.Book, error)` | Read existing, merge nilable fields, validate progress, recompute `completed_at` (set when `current >= total`, clear when `current < total`). |
| `deleteBook` | `DeleteBook(id string) error` | |
| `markBookComplete` | `MarkBookComplete(id string) (model.Book, error)` | `current_progress = total_progress`, `completed_at = now()`. |
| `getSettings` | `GetSettings() (model.UserSettings, error)` | Fall back to defaults (`start=8`, `end=22`, `stats_start_date=nil`) if row missing. |
| `updateSettings` | `UpdateSettings(req model.UpdateSettingsRequest) (model.UserSettings, error)` | Merge nilable fields, validate, then `UPDATE`. |

## Test plan (per method, before implementing)

- Table-driven where sensible (e.g. `UpdateBook` with/without `completed_at` transitions).
- Use `t.TempDir()` for file-backed variants; `:memory:` for the fast happy path.
- Each CRUD also gets one file-backed test to surface driver-level issues.
- Add entries to `wails/store/TESTS.md` and `wails/service/TESTS.md` as tests are added.

## Guardrails

- **Do not** touch frontend files. The `invoke("…")` call sites still reference the old Tauri command names; prompt 05 rewires them.
- **Do not** change the DB schema. Any mismatch between `model/` and `schema.sql` is a bug in `model/`, not the schema.
- Use `time.Now().UTC().Format(time.RFC3339)` for timestamps so values match what the existing Tauri backend wrote.
- Wrap errors with context: `fmt.Errorf("insert book %s: %w", id, err)`.
- Never ignore an error. Close rows explicitly.
